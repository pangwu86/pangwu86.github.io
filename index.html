iOS踩坑记录001-误删Assets.xcassets后如何恢复

发表于 2017-09-01 | | 阅读次数

删除后，Xcode没有undo操作，就从回收站里手动copy出来放回原位，但是编译后发现图标的设置不起作用了，但也没有相关错误提示。后来经过测试发现需要2个步骤

  1. 删除Xcode中看到那个文件，选择“Remove Refrence”，也就是删除引用
  2. 在项目文件夹上右键，选择“Add Files to Project”，选择刚才的文件夹

这样再编译就正常了。

群辉NAS实用技巧-Docker安装远程迅雷

发表于 2017-07-17 | | 阅读次数

迅雷可谓是国人最常用也是最爱的下载工具，而NAS上的远程迅雷更是可以让你出门在外一样控制下载方便极了。

安装教程网上一片一片的，这里就不再重复了，这里贴2个me之前参考的

  • 群晖 NAS DSM 系统，只要三步使用 Docker 安装迅雷远程下载
  • 黑群晖从装机到使用全过程 篇三：迅雷远程下载和架设个人网站

下面重点说一下2017.07迅雷发出公告后，无法使用远程迅雷后的一个解决方案

后来经过大家研究发现是Docker镜像中的Xware1.0.31_x86_32_glibc授权过期了，而较早版本的还可以用，比如Xware1.0.20_x86_32_glibc或Xware1.0.21_x86_32_glibc

所以解决方案很简单，就是替换成旧版本。不过手动替换需要一定的linux基本知识，还是有一定门槛的。

大概步骤如下：

  1. 下载老版本Xware，1.0.20固件连接 密码：3tvi
  2. 停止当前Docker容器，重新启动但不要启动Xware，否则无法替换正在使用的文件
  3. 替换文件为 /xware/bin/portal与/xware/bin/lib/*所有文件
  4. 重启Docker，应该就能恢复正常了
