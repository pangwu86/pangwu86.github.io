nutzam更新

发表于 2017-01-08 | | 阅读次数

先吐槽下

这周被派遣了一个重要任务，那就是把nutzam的页面适配手机版，打开看了一下，网页版效果还是不错的，切换到手机上发现还是那个页面效果尚可接受，然后切换到文档页就发现没法看了，一行一个字，页面大部分内容被侧边栏的导航给占据了，所以明显这个在实现的时候没有考虑手机去查看

首页，明明下面没东西了，竟然还有滚动条？！

文档页面，手机直接没法看了… 上面的菜单也看不到了…

修改计划

本来想着仅仅在现有基础上修改，后来跟peter讨论了下还是直接重写一版吧，因为不仅仅是文档页没有适配手机，主页也是没有适配的，所以手机查看的体验还是不好

那就没啥可说的了，重写呗

文档编译步骤

这里说的是老版本的首页，首先需要clone几个项目，nutz，nutam，然后编译通过zdoc2,

修改pages/index.xml，然后将nutz-doc指向本地的目录

在bash下执行编译命令，如果成功就会生成静态网页到指定目录了

PS:当前版本的需要启动一个web服务来访问，否者文档页的侧边栏是显示不出来的

修改点与技术选型

现在的文档是通过zdoc进行编译生成的，所以有些页面或布局模板的使用还需遵守zdoc的规定
为了比较快的实现，决定先用bs4实现一版出来

文档页的侧边栏是zdoc生成的tree.html,这里为了能够方便页面调用需求修改zdoc再输出一份tree.json，也就是仅仅包含侧边栏信息的纯js对象数据

1天过后

中途精力了两场灾难

  1. bs4是用dropmenu时少了一个属性，以为没用的，结果除了ios设备都正常，但ios上死活点不出来，写了半天了发现竟然不能用咋办？换bs3。然后写着写着又发现是自己的问题，再换回bs4…..
  2. git提交后checkout了老版本，然后忘了开branch，结果写完后一个误操作回到了checkout前的状态….幸好用mac有time machine，找回了一个不那么久远的版本也算是捡了一命了，否者吭哧吭哧写了半天东西又得重写一遍…

总之，这次事故得到经验就是

一定要好好的读文档做实验，多方面确认后再下决定，否者就白费力气
一定要养成多提交的好习惯，再就是备份很重要，time machine救命神器
