关于<<编写高质量的代码>>

发表于 2015-01-27 | | 阅读次数

豆瓣链接

首先这是一本2010年出版的前端技术书, 现在才看说实话有点晚了. 毕竟前端圈的技术更新快, 里面的部分知识点已经out了.

整本书总体分为两部分

  1. 作者的经验
  2. 前端知识点

作者首先对前端开发做了个总结, 那就是”精一行, 通十行”. 主要原因就是前端技术界限不明显，各个领域相互渗透.

想做出一个画面精美, 布局合理, 交互体验流畅的页面, 需要对js, html, css的运用恰到好处才行, 所谓恰到好处就是指实现一个功能你知道应该怎么组合这几种语言来做到最优实现. 当然想要更完美, 你可能还需要懂一点配色原理, 甚至能自己PS一下图片, 对于后台技术与后台做接口对接要有一定的认识.

毕竟对于前端界面来说, 要实现的不仅仅是”能用”而是”好用”.

作者把一下编程常见思路也代入了前端开发, 这些东西在2010年前后应该说是各种语言都在说的事情

  • 必要的注释
  • 提供代码重用性
  • 注意兼容性

当然由于前端本身缺少一套事实上的工作流程标准(对比java, 2010年的时候ssh就是标配呀), 所以前端在架构设计, 流程规范中始终处于自己搞一套或者借鉴某大厂的一套. 当然前端各种代码混合工作的特性, 也算有利有弊, 没有标准, 才能百花齐放么.

本书共5章, 前2章是作者经验, 后3章分别对html, css, js做了深入的讨论, 当然很多知识点你google也能找出一堆教程.

总的来说, 对于前端初中级开发人员, 看一遍总是会有收获的.

摘抄笔记

========================================== 关于前端开发

前端界限不明显，各个领域相互渗透, 像配色，布局，交互等等

精一行，得通十行

  1. css布局是前端开发的基本功
  2. 会用原生js, 熟悉常用框架，最佳实践
  3. 了解后台语言
  4. 兼容主流浏览器

========================================== 关于html与CSS

了解标签的语义，为什么有这些标签，什么时候用，优点是什么

css布局，通过css控制页面中元素样式，位置，大小，颜色等

去掉样式，看网页结构是否组织良好有序，是否仍有很好的可读性

什么是怪异模式?

如何组织css, base, common, page

关于base.css(reset.css)

相邻的margin-left, margin-right不会重合, 但是相邻的margin-top, margin-bottom会重合

最好不要混用margin-top跟margin-bottom,

css的权重, html标签1, class是10, id是100

  1. 权重高的来设置样式
  2. 权重一样, 按照后定义的

CSS sprite
优点 减少http请求
缺点 合并的图片仅仅适用于背景图片 img src不能用
维护麻烦, 不宜修改

CSS hack
大部分针对IE
<!–[if IE 6]>

<![endif]–>

选择符前缀法
html .test // only IE6 +html .test // only IE7

样式属性前缀发
_ IE6生效 *IE6,7生效

超链接访问后hover不出现
a:visited a:hover顺序要注意, 不要放错

a标签四种状态, 排序最好是 love hate原则, l(link)ov(visited)e h(hover)a(active)te

块级元素, 不论width都占一行

行内元素, 一行满了才换行, 宽度随内容儿变化

行内元素的margin跟padding要注意, 水平方向正常, 但是垂直方向的不会产生边距效果

display的 block, inline, inline-block(IE6,7不支持)

其中inline-block, 块级元素, margin跟padding正常, 宽度不占满父元素, 可以和其他行内元素在同一行里

hasLayout属性

改变文档流

position:relative position:absolute 可以激活元素的left, top, bottom, right 和z-index属性

z-index: 0 默认

float 只能设置left, right, 但依旧在z-index:0

设置了float跟absolute的元素, 都以 inline-block 形式显示

水平居中

行内元素, 在父元素使用 text-align:center, 文字图片都会水平居中

块级元素
1.确定width, margin-right:auto, margin-left:auto
2.不定width
2.1 使用table, table宽度由内部元素撑起, 然后设定margin即可
2.2 设置line, 父元素text-align:center, 但是这样就无法设置宽度了
2.3 父元素float relative left:50% 子元素float relative left-50%

垂直居中

  1. 不定height, 设置上下padding
  2. 确定height, 设置line-height跟height一样大
  3. 多行文本
    3.1 vertical-align 只有父元素为td或th时起作用, 设置父元素的display:table-cell(not IE6, 7), 激活vertical-align 或者直接上table

========================================== 关于js

避免全局变量泛滥, 合理使用命名空间, 为代码添加必要的注释

DOMReady比window.onload更适合用来调用初始化函数, 因为onload要在所有元素加载完后才触发(比如图片, flash等很大的文件)

冒泡事件处理
IE: e.cancelBubble = true;
FF: e.stopPropagation()

事件监听 on XXX 后面的会覆盖前面的, 使用 attachEvent 跟 addEventListener 可以叠加事件

节点属性 通过 node.XXX, 如果使用 node.getAttribute(‘XXX’) IE与FF结果可能不一致

但是自定义属性只能通过 node.getAttribute(‘XXX’) 获得

Event对象

IE中作为window的一个属性, 全局作用域下
FF中作为事件的参数存在
