对于前端开发的一点思考

发表于 2014-07-02 | | 阅读次数

最近应该说前端开发方面进入了一个瓶颈期, 不得不停下脚步好好想想.

从正式开始接触前端到现在大概有2年半了, 从html标签根部不懂几个, 到后来可以写大个头的jquery控件, 其实时间并不长.

但一旦上手了一个模式以后, 就陷入了一个重复循环的模式, 经常做着大量重复性的劳动性工作. 最经常写的就是各种业务逻辑的UI控件, 虽然手熟后写起来很快, 但是总是有那么一点点烦人的说.

总的来说, 应该还是”这么写是不是最好的方法”这样的问题在不停的困惑着自己

从extjs的封装, 到jquery控件的编写, 到nutz-ui项目的建立, 再到octopus里bootstrap的尝试, 一次次的摸索, 总觉得还是差那么一点

静下心来决定从0开始, 从最简单的util方法开始总结吧, 把手头用的到的先整合起来

花了几天时间看了看最新的前端咨询, 各种新框架新工具的使用, 又是一堆东东呀

不过有几点还是有启发的

  • 控件间的依赖管理
  • 模型与视图的自动绑定
  • css的加强,sass或less
  • 自动化工具的使用
  • 自定义事件
  • MVVM模型

一个自己用起来顺手又符合大众潮流的前端框架或者说前端工作流程真是需要好好搞一下

这次准备把nutz-ui重新写一下, 开了新的branch, 看看能不能搞的更像样一点
