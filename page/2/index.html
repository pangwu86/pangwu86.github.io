如何实现读取本地图片自动上传服务器

发表于 2017-03-23 | | 阅读次数

目前有两个思路，都需要一个跳转服务器来支持

第一种：

  1. 本地的启动web server上传读取的图片到walnut
  2. 上传后使用walnut的websocket通知机制把刚刚上传的图片的id发到客户端
  3. 浏览器窗口使用websocket把接收到的id记录下，然后图片展示那显示出来

第二种：

  1. 本地启动web server对外提供读取本地图片能力
  2. 浏览器窗口ajax读取web server的图片，然后将图片内容直接转发到walnut上，获取文件id，随后展示出来

第二种的优点就是过程是由浏览器触发的，不需要ws长连接，防止了ws多条链接一起收到内容的问题

BufferedImage选择图片类型

发表于 2017-03-04 | | 阅读次数

之前为Nutz添加了一个根据文字生成头像的方法Avatar，但遇到了保存为jpg格式生成图片不对的bug。

后来发现是在新建BufferedImage时设置的参数问题：

也就是默认选择了带有Alpha通道的格式，而jpg是没有的所以生成结果有了问题。

本来改成了用户可以手动设定BufferedImage类型，但后来发现其实可以通过用户设定文字颜色与背景色进行判断，如果是带有Alpha通道的比如rgba那就应该用BufferedImage.TYPE_INT_ARGB，否者就改成BufferedImage.TYPE_INT_RGB

java代码可以通过Color去判断Alpha值，这就可以自动设定正确的类型了。
